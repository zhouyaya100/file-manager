## 🔧 Bug 修复详情

### 2026-03-15 修复记录

#### 1. 权限撤销功能报错"参数不完整"
**问题**: 点击权限管理页面的"撤销"按钮提示失败  
**原因**: 前端发送 `permission_id`，后端期望 `directory_path`  
**解决**: 修改前端请求参数，同时更新函数签名和按钮传参

#### 2. 路径管理无法删除路径
**问题**: 点击删除按钮提示成功，但刷新后路径还在  
**原因**: HTML `onclick` 属性中 Windows 路径的反斜杠被浏览器转义（`D:\AMD` → `D:AMD`）  
**解决**: 改用 `data-*` 属性 + 事件监听器模式

#### 3. 路径格式不统一导致重复记录
**问题**: 同一路径显示多次，分配权限提示"权限级别未变化"  
**原因**: `D:/` 和 `D:\` 在数据库中被视为不同路径  
**解决**: 在路径注册和权限分配接口添加 `os.path.normpath()` 标准化

#### 4. 目录卡片点击无法进入
**问题**: 点击目录选择器卡片后直接弹回仪表板  
**原因**: 
- 目录选择器元素缺少 ID
- `resolve_user_path()` 变量名错误
- `accessible_dirs` 返回字典列表而非字符串列表  
**解决**: 添加 ID、修复变量名、修正返回类型

#### 5. 权限记录乱码
**问题**: 权限记录的 directory_name 显示为乱码  
**原因**: 测试时编码不一致  
**解决**: 清理旧记录，重新分配
